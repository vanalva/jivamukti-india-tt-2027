Body text — three sizes

large — intro paragraphs

Our faculty of senior Jivamukti Yoga Teachers are pleased to offer their wisdom & knowledge in the land of its source.

main — body copy

Bhakti is the yoga of devotion, which takes the energetic science of yoga from practice into an experience of transcendental joy.

ClassSize (range)Use forDo not use for
u-text-style-large 16px → 22px The first paragraph under a section header — the "intro" sentence that sets context before the main grid or list. One per section at most. Body copy inside cards, accordions, or repeated list items.
u-text-style-main 14px → 20px All body paragraphs: accordion content, profile rows (Bhakti section), tenet descriptions, accordion content, housing intro, travel info. Section intro (use large). UI labels (use small).
u-text-style-small 12px → 14px Nav banner text, amenity labels, legal/fine print, caption text, any UI label that shouldn't compete with body copy. Running paragraph text — it will feel too small at body length.

Typography rule

Never use u-text-style-large twice in the same section. It is a spotlight — one per section. If two paragraphs both feel important, the first is large, the second switches to main.

On the page copy: do not rewrite or shorten existing copy. The client has confirmed it should stay as-is unless they explicitly provide new copy.

02 Line height rules

Line heights are pre-assigned by the text style classes. Do not override them unless there is a specific layout reason.

TokenValueUsed by
--_typography---line-height--small 1.2 Display, H1, H2 headings. Tight because large type needs less leading.
--_typography---line-height--medium 1.3 H3, H4 headings.
--_typography---line-height--large 1.4 H5, H6, u-text-style-large, u-text-style-small.
--_typography---line-height--huge 1.6 u-text-style-main — body text at reading length needs the most leading for comfort.

Line height rule

Use line-height--huge (1.6) only for running body text. Use line-height--large (1.4) for anything under 3 lines — accordions, labels, captions. Headings always get small or medium — never 1.6 on a heading.

03 Section anatomy — the 3-layer structure

Standard content sections follow a strict 3-layer nesting. Understanding this structure is the most important pattern to get right — every spacing issue on the page comes from breaking it.

<section class="u-section">

Adds top + bottom padding via --_spacing---section-space--main-112px (≈64px → 112px fluid). This is the breathing room between sections.

<div class="u-container">

Constrains width to 90rem max and adds horizontal page margins. Everything goes inside this.

<div class="u-content-wrapper"> (optional — only when there's a section header)

Max-width 44rem. Holds the section_divider + h2 + intro u-text-style-large paragraph. Add is-centered for centered alignment (see §6).

Content grid / list / component

The actual content: faculty grid, accordion list, housing rooms, prereq cards, etc. Always a sibling of u-content-wrapper, never nested inside it.

When to use u-content-wrapper

Use it whenever the section has a header block (divider + h2 + optional intro paragraph). Skip it when the section heading is styled with its own custom class (e.g. india_bhakti_heading, india_housing_heading) or the section has no visible header (video section, marquee).

Custom-layout sections (no u-section wrapper)

Four sections do NOT use u-section because they have unique full-bleed or split layouts: Hero, Karmabhumi / Land of Action, Application Requirements, and Registration Dates. These manage their own spacing internally. Do not add u-section to them.

The section_divider

A thin horizontal rule that always appears immediately before the h2 heading. It is a visual signal that a new section is starting. It is never skipped when a u-content-wrapper is present.

Markup

<div class="u-content-wrapper">
  <div class="section_divider"></div>
  <h2 class="u-heading" data-style="h2" id="section-id">Section Title</h2>
  <p class="u-text-style-large">Intro paragraph (optional).</p>
</div>

04 Spacing — when to use which token

All spacing uses CSS variables. Never hardcode pixel or rem values. Here are the tokens in use on this page and what they're for:

TokenRangeUse for
--section-space--main-112px 64px → 112px Applied automatically by u-section. Section top/bottom padding. Do not set manually.
--section-space--small-80px 48px → 80px Compact sections where the full main spacing would feel excessive. Not currently used on this page but available if a new tight section is added.
--_spacing---space--8-64px 40px → 64px Gap between the u-content-wrapper and the content grid/list below it. Row gap in the faculty grid.
--_spacing---space--7-48px 36px → 48px Margin below the u-content-wrapper. Gap between major blocks within a section.
--_spacing---space--6-40px 32px → 40px Card internal top/bottom padding. Generous internal spacing.
--_spacing---space--5-32px 28px → 32px Button horizontal padding. Section panel internal padding (land panel, pricing card).
--_spacing---space--4-24px 20px → 24px Default margin between paragraphs and text elements. Gap between card columns.
--_spacing---space--3-16px 14px → 16px Internal card padding (tight). Nav link gaps. Dropdown item padding.
--_spacing---space--2-12px 10px → 12px Tight gaps: between divider and heading, between heading and badge/role label.
--_spacing---space--1-8px 6px → 8px Micro gaps: icon-to-text, amenity list item spacing, nav banner padding.

Spacing rule

Move up one step when something feels tight. Move down one step when something feels too spread out. Never skip more than one step in a single transition — e.g. do not go from space--3 directly to space--6 for consecutive elements in the same list.

05 Theme variants — light, brand, dark, contact

This project has four distinct surface colors. Adding the right theme class to a section wrapper changes background, text, border, and button colors automatically — you do not need to set colors individually.

Default

u-theme-light

Warm peach (#efd8c9). Used on <body> — every standard section inherits this unless overridden. Most of the page is this color.

Brand

u-theme-brand

Deep burgundy (#541820). Used on hero left panel, testimonials section. Apply to the section element or any wrapper — all descendant text/buttons/cards adapt automatically.

Dark

u-theme-dark

Near-black (#1a1a1a). Used on the footer. Do not use on content sections above the footer.

Contact

india_contact_section

Bright Jivamukti red (#eb292b). Applied via india_contact_section class — not a LUMOS theme class, but an override block in the CSS embed. Used only for the Contact/Form section.

Theme rule

Never set color, background-color, or border colors directly in a new custom class. Use a theme class on the ancestor and let it cascade. If you need a one-off override (e.g. the gold nav banner), use an existing specific class like is-gold — do not create a new inline style.

Gold / secondary accent

The warm gold (--swatch--secondary-500, #daab85) is used sparingly: the nav announcement banner (is-gold), button overrides inside u-theme-brand sections. Do not introduce gold as a background for new sections — it's an accent only.

06 Alignment — centered vs. left-aligned

All sections default to left-aligned text. Use is-centered only in specific cases.

ClassWhen to useCurrently on
u-content-wrapper Default. Section header block: left-aligned text, divider flush left. Faculty, Travel, Gallery, Testimonials, Video testimonials sections.
u-content-wrapper is-centered When the section content below is symmetric (a centered grid, a price table, a swiper that spans full width). The wrapper text and divider both center. Tuition & Fees section (pricing grid is symmetric).

Alignment rule

Default to left. Ask: "is the content below symmetric?" — symmetric = equal columns, centered carousel, single CTA. If yes: is-centered. Asymmetric grids, lists, and image+text rows always stay left.

profile_layout — alternating image/text rows

The Bhakti section uses profile_layout for alternating rows (image left / text right). Add is-reversed to flip the order for even-numbered rows. This creates visual rhythm without changing the HTML reading order.

Pattern

<!-- Row 1: image left, text right -->
<div class="profile_layout"> ... </div>

<!-- Row 2: text left, image right -->
<div class="profile_layout is-reversed"> ... </div>

<!-- Row 3: image left, text right again -->
<div class="profile_layout"> ... </div>

07 Buttons — primary vs. secondary

Live preview

Apply Now Learn More
ClassVisualUse forDo not use for
button_primary Filled burgundy, cream text, pill shape The single most important CTA on a section — always "Apply Now" or equivalent direct action. Maximum one per section. Secondary actions, navigation links, or repeated CTAs that aren't the main action.
button_secondary Transparent with border, dark text, pill shape Secondary actions: "Join Us", "Learn More", "View All". Can appear alongside a primary button or alone. Never as the only button on a section where a strong CTA is needed — primary should lead.

Button rule

Buttons inherit color from their theme context. Inside u-theme-brand (burgundy), buttons automatically switch to gold/cream. Inside india_contact_section (red), they switch to cream/red. You do not need to add extra classes — just use button_primary or button_secondary and the theme does the rest.

Always use <a> for navigation CTAs and <button type="button"> for JS-triggered actions (tab switching, accordion toggles).

08 Component patterns

Faculty card — faculty_card

Used in the Faculty section. 5-column grid on desktop, 3 on tablet, 2 on mobile. Each card: photo → thin divider → body (role label + name + credential).

Structure

<div class="faculty_card">
  <img src="../assets/[photo]" alt="[Name]" class="faculty_card_photo" loading="lazy">
  <div class="faculty_card_divider"></div>
  <div class="faculty_card_body">
    <div class="faculty_card_role">[Role e.g. Facilitator]</div>
    <div class="faculty_card_name">[Full Name]</div>
    <div class="faculty_card_credential">[e.g. Advanced Certified]</div>
  </div>
</div>
Photo aspect ratio is square (1:1). Crop images to square before using. The faculty_card_photo class enforces this with object-fit: cover.

Tenet tabs — tenets_tab_wrap

Interactive tab component for the Five Tenets section. Left column: tab buttons. Right column: content panels (all stacked, only one visible at a time via is-active class). The JS in the page handles switching.

Key rule

Each panel uses data-style="h2" for the tenet name (large, decorative) and u-text-style-main for the description. The Sanskrit subtitle uses tenets_content_sanskrit — do not change its class to a heading style; it is not a navigation heading.

There are exactly 5 tabs: Ahimsa, Nada, Bhakti, Dhyana, Shastra. Do not add or remove tabs without updating both the tab button list and the panel list.

Accordion — accordion_component

Used in the Travel Info section. Each accordion has a toggle button (always h5-level heading inside) and a collapsible content area. JS handles the open/close via aria-expanded.

Structure

<div class="accordion_component">
  <div class="accordion_toggle_button" role="button" tabindex="0" aria-expanded="false">
    <h3 class="accordion_toggle_heading u-heading" data-style="h5">[Title]</h3>
    <svg class="accordion_toggle_icon" ...></svg>
  </div>
  <div class="accordion_content_wrap">
    <div class="accordion_content_padding">
      <p class="u-text-style-main">[Content]</p>
    </div>
  </div>
</div>
All body text inside accordion panels uses u-text-style-main. Do not use large inside a collapsed panel — the user hasn't opened it yet, so it is not the "intro" of a section.

Prerequisites card — india_prereq_card

Three-column grid. Each card: title (its own india_prereq_card_title class — not a heading style, it uses the project's own sizing) → divider → book list (each item is a india_prereq_book_item paragraph) → CTA button at bottom.

Copy rule

The reading list copy is exact and must not be changed — it is official program content. If an item is updated, use the exact text provided by the client, including italics/capitalization.

Testimonial — testimonial_card

Lives inside a Swiper carousel on the burgundy (u-theme-brand) section. On brand backgrounds the white fill is removed automatically — see styleguide §Card surfaces on u-theme-brand. Always include the checkmark image and a name/location attribution.

Pricing card — india_pricing_card

Two cards side by side on a light section. The featured card (Early Bird) gets the is-featured combo class — filled burgundy background, inverted text. The standard card stays transparent with a burgundy border. Do not apply is-featured to more than one card.
